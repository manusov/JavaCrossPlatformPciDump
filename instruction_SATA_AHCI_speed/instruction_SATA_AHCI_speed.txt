Исследование SATA AHCI (Serial ATA Advanced Host Controller Interface).
Используется утилита RWEverything.
Плата ASUS TUF Gaming Z590-Plus WiFi.
Определяется скоростной режим подключенных устройств (SATA Gen1/Gen2/Gen3).
Нумерация соответствует скрин-шотам step_01 - step_11.
-----------------------------------------------------------------------------

   ПЕРЕД НАЧАЛОМ ИССЛЕДОВАНИЙ РЕКОМЕНДУЕТСЯ СОХРАНИТЬ ВАЖНУЮ ИНФОРМАЦИЮ.

-----------------------------------------------------------------------------

1) Выбираем контроллер SATA AHCI в списке PCI устройств.

2) Определяем базовый адрес диапазона memory-mapped I/O для контроллера SATA AHCI.

3) Определяем количество поддерживаемых портов, а также битовую маску, выделяющую
   подмножество портов системной логики, используемое на конкретной платформе.

4) Проверяем порт 0. Регистровые блоки, ассоциированные с конкретными портами,
   расположены начиная со смещения 0100h относительно базового адреса AHCI MMIO.
   Каждый блок занимает 128 байт = 80h. Таким образом:
   Port 0 register block base = A1E02000h + 100h = A1E02100h.
   Port 1 register block base = A1E02000h + 180h = A1E02180h.
   Port 2 register block base = A1E02000h + 200h = A1E02200h.
   Port 3 register block base = A1E02000h + 280h = A1E02280h.
   Port 4 register block base = A1E02000h + 300h = A1E02300h.
   Port 5 register block base = A1E02000h + 380h = A1E02380h.
   Базовый адрес AHCI MMIO зависит от реализации конкретной платформы и ее firmware.
   Значение A1E02000h только для данного примера. См. шаг 1 для определения адреса.
   Регистр статуса порта 0 по смещению 28h, A1E02100h + 28h = A1E02128h,
   указывает на то, что к порту 0 не подключено устройство.

5) Аналогично, к порту 1 не подключено устройство.

6) К порту 2 подключено устройство в режиме SATA Gen 3 (6 Gbps), 
   передача данных не выполняется в этот момент.

7) К порту 3 подключено устройство в режиме SATA Gen 3 (6 Gbps), 
   передача данных выполняется в этот момент.

8) К порту 4 не подключено устройство.

9) К порту 5 не подключено устройство.

10) Перепроверим порт 2 во время копирования данных на жестком диске.
    Статус изменился и показывает, что выполняется передача.

11) Передача выполняется и для порта 3. Хотя на этом порте передача выполнялась и
    ранее, что зафиксировано на шаге 7. Предположительно это результат фоновых
    процессов в ОС на момент проверки статуса. В общем случае, для получения
    достоверного результата рекомендуется запустить на SATA-устройстве,
    подключенном к исследуемому порту достаточно продолжительную операцию
    копирования файлов. Иначе, энергосберегающие функции могут изменить состояние
    порта, в том числе понизить скорость обмена.
